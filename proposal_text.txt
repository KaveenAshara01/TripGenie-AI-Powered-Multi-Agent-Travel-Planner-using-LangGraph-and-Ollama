Project Title  
TripGenie: A Locally Hosted Multi -Agent Travel Planning System Using LangGraph and 
Ollama  
 
Problem Statement  
Planning a trip can be time -consuming and complicated. Travele rs must research 
accommodation, estimate expenses, discover attractions, and organize an itinerary. This often 
requires switching between multiple platforms and manually consolidating information.  
TripGenie solves this problem by using an autonomous team o f AI agents that collaboratively 
generate a complete travel plan based on user preferences. The system operates entirely on the 
user's local machine, ensuring zero cloud costs, data privacy, and compliance with assignment 
requirements.  
Unlike a conventiona l chatbot, TripGenie decomposes the planning process into specialized tasks 
handled by multiple intelligent agents, each equipped with dedicated tools and responsibilities.

System Overview  
TripGenie accepts a natural language travel request such as:  
"Plan a 4 -day budget -friendly trip to Ella for two people with adventure activities."  
The system then autonomously:  
1. Extracts trip requirements  
2. Calculates estimated expenses  
3. Finds suitable accommodations  
4. Recommends activities  
5. Generates a detailed day -by-day itinerary  
 
Core Technologies  
• Orchestration Framework:  LangGraph   
• Local LLM Runtime:  Ollama   
• Language Mo del: Llama 3   
• Programming Language:  Python 3.11+  
• Testing Framework:  Pytest  
• Observability:  Python Logging  
• Data Storage:  Local JSON/CSV datasets  
 
Multi -Agent Architecture  
The system consists of four specialized agents.  
 
1. Coordinator Agent  
Responsibilities  
• Accept user input  
• Extract structured trip details  
• Validate inputs  • Initialize global state  
• Route tasks to downstream agents  
Example Input  
"Plan a 3 -day luxury trip to Kandy for 2 people."  
Example Output  
{ 
  "destination": "Kandy", 
  "days": 3, 
  "travelers": 2, 
  "budget_type": "luxury"  
} 
Tool Used  
• Natural language parser  
 
2. Budget Agent  
Responsibilities  
• Estimate total trip cost  
• Calculate accommodation, food, transport, and activity expenses  
• Suggest budget breakdown  
Tool Used  
• calculate_trip_budget()   
Example Output  
{ 
  "total_cost": 85000, 
  "hotel_cost": 45000, 
  "food_cost": 18000, 
  "transport_cost": 12000, 
  "activities_cost": 10000 
} 
 
3. Accommodation Agent  Responsibilities  
• Find hotels matching budget and destination  
• Rank accommodations  
• Recommend the best options  
Tool Used  
• search_hotels()   
Data Source  
• Local JSON hotel dataset  
 
4. Itinerary Agent  
Responsibilities  
• Recommend attractions and activities  
• Generate a day -by-day travel plan  
• Combine all previ ous outputs  
Tools Used  
• get_local_activities()   
• generate_daily_schedule()

Global State Management  
LangGraph maintains a shared state object across all nodes.  
from typing import TypedDict , List, Dict 
 
class TravelState (TypedDict ): 
    user_request : str 
    destination : str 
    days: int 
    travelers : int 
    budget_type : str 
    budget: Dict 
    hotels: List[Dict] 
    activities : List[Dict] 
    itinerary : str 
    logs: List[str] 
Every agent reads from and writes to this state.  
 
Custom Tools  
Each team member should implement o ne tool.  
 
Tool 1: Budget Calculator  
def calculate_trip_budget ( 
    destination : str, 
    days: int, 
    travelers : int, 
    budget_type : str 
) -> dict: 
Calculates estimated travel expenses.  
 
Tool 2: Hotel Search  def search_hotels ( 
    destination : str, 
    budget_type : str 
) -> list: 
Returns recommended accommodations.  
 
Tool 3: Activity Finder  
def get_local_activities ( 
    destination : str 
) -> list: 
Provides destination -specific attractions.  
 
Tool 4: Schedule Generator  
def generate_daily_schedule ( 
    days: int, 
    activities : list 
) -> str: 
Creates a detailed itinerary.
