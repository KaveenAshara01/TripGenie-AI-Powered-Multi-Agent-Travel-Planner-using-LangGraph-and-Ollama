Project Overview  
Your objective is to design, build, and deploy a locally -hosted Multi -Agent System (MAS) that 
automates a complex, multi -step problem.  
You have complete freedom to choose the industry vertical or problem domain for your project. 
Whether you choose to automate a process in Finance, Healthcare, E -Commerce, Education, 
Entertainment, or internal Software Engineering workflows, your system wil l be graded strictly on 
how well it implements the core architectural components of Agentic AI.  
Rather than building a generic conversational chatbot, your system must function as an 
autonomous team of agents that can perceive a problem, reason about it, use tools to interact 
with the environment, and synthesize a concrete solution.  
 
Technical Constraints (Zero -Cost & Local)  
To ensure data privacy and zero cloud costs, this system must run entirely on your local machines:  
• The LLM Engine:  You must use local Small Language Models (SLMs) via Ollama  (e.g., 
llama3:8b, phi3, qwen).  
• The Orchestrator:  Use an open -source framework like LangGraph , CrewAI , or AutoGen  to 
manage the state and routing.  
• Execution:  The system must run locally without requiring paid API keys (OpenAI, 
Anthropic, etc., are strictly prohibited for this assignment).  
 
 
 Development Guidelines: What the System Must Have  
Regardless of the domain you choose, your final Multi -Agent System must demonstrate the 
following architectural components:  
1. Multi -Agent Orchestration:  The system must utilize at least 3 to 4 distinct agents 
interacting with one another (e.g., utilizing a Coordinator -Worker -Delegator or sequential 
pipeline model).  
2. Tool Usage:  Agents cannot rely solely on their internal LLM knowledge. The system must 
include custom Python tools that allow agents to interact with the real world (e.g., 
reading/writing local files, executing terminal commands, querying a local database, or 
calling  free public APIs).  
3. State Management:  The system must demonstrate how "global state" (data) is securely 
passed from one agent to the next without losing context.  
4. LLMOps/ AgentOps & Observability:  You must implement a logging or tracing mechanism 
to record the specific inputs, tool calls, and outputs of the agents during execution.  
 
Individual Requirements: What Every Student Must Do  
To ensure all team members are exposed to the full spectrum of Agentic AI development, there 
are no "project managers" or "documentation only" roles. Every individual student in the group 
is required to:  
1. Build an Agent:  Design the system prompt, constraints, and persona for at least one 
specific agent in the swarm.  
2. Build a Tool:  Write at least one custom Python tool (with strict type hinting and docstrings) 
that your specific agent will use.  
3. Implement Testing/Evaluation:  Write an automated evaluation script (e.g., property -
based testing or an "LLM -as-a-Judge" test) to validate that your specific agent's output is 
accurate and secure.
